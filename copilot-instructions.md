# GitHub Copilot Instructions

---
ApplyTo: **
---

1. install GitHub Copilot extension in your code editor.
2. Sign in to GitHub and enable Copilot for your account.
3. Full react project structure with components, hooks, context, and styles setup.
4. Project must be using JS and Tailwind CSS for styling. Install Tailwindcss and stop using postcss completely.
5. Use functional components and React hooks.
6. Ensure code is clean, well-commented, and follows best practices.
7. Start typing code or comments to get relevant suggestions from Copilot.
8. Review, modify, and test the suggestions as needed to fit your project requirements.
9. Save your files to see Copilot's suggestions in action.
10. Provide feedback to GitHub Copilot to help improve its suggestions over time.
11. Regularly update the Copilot extension to access the latest features and improvements.
12. Explore Copilot Labs for experimental features and enhancements.
13. Use Copilot's code completion and generation features to speed up development.
14. Leverage Copilot's ability to generate tests and documentation for your code.
15. Apply input validation and error handling in your components.
16. set up my node so I can run the "npm run dev" command to see my work.
17. Ensure that the project is configured to use Vite as the build tool.
18. Always start my style document with the following line: 
```
css@tailwind base;
@tailwind components;
@tailwind utilities;

*{
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
```
19. Install necessary dependencies for Tailwind CSS and configure it properly in the project.
20. Make sure that all components are responsive and mobile-friendly.
21. Use meaningful names for components, functions, and variables to enhance code readability.
22. Regularly commit changes to version control with clear and descriptive commit messages.
23. Validate forms and user inputs with zod schema validation.
24. Set up routing in the project using React Router for navigation between different pages.
25. Always use the most AI-appropriate language model available for generating code suggestions. 

26. Ensure that the project structure is organized and follows a logical hierarchy for easy navigation.
